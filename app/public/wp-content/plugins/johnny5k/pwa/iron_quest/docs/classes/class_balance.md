# Cross-Class Balance Framework (v1)

## Overview

Each class in IronQuest is designed to be:

- Equally viable
- Differently optimal
- Appealing to different user types

No class should be:

- Strictly stronger
- Required for success
- Dominant across all systems

## Core Balance Philosophy

Each class owns a primary advantage domain:

| Class   | Primary Advantage       | Secondary Advantage      |
| ------- | ----------------------- | ------------------------ |
| Warrior | Strength / Intensity    | Boss dominance           |
| Ranger  | Movement / Travel       | Cardio efficiency        |
| Mage    | Consistency / Recovery  | Error forgiveness        |
| Rogue   | Accessibility / Economy | Short-session efficiency |

## System Ownership (Who Wins Where)

### XP Generation

| Scenario        | Best Class |
| --------------- | ---------- |
| Heavy lifting   | Warrior    |
| Cardio / steps  | Ranger     |
| Full completion | Mage       |
| Short workouts  | Rogue      |

### Gold Economy

| Scenario            | Best Class |
| ------------------- | ---------- |
| Raw gold gain       | Rogue      |
| Boss rewards        | Warrior    |
| Location completion | Ranger     |
| Consistent farming  | Mage       |

### Boss Performance

| Scenario                     | Best Class |
| ---------------------------- | ---------- |
| High performance / dominance | Warrior    |
| Prepared / cardio-supported  | Ranger     |
| Clean completion             | Mage       |
| Efficient / low-risk runs    | Rogue      |

### Survival / HP Management

| Scenario                        | Best Class |
| ------------------------------- | ---------- |
| Raw HP pool                     | Warrior    |
| Movement pacing                 | Ranger     |
| Recovery / healing              | Mage       |
| Forgiveness / reduced penalties | Rogue      |

### Accessibility

| Scenario         | Best Class |
| ---------------- | ---------- |
| Beginners        | Rogue      |
| Habit builders   | Mage       |
| Active users     | Ranger     |
| Advanced lifters | Warrior    |

## Hard Balance Rules (Do Not Break These)

### Rule 1: No Class Dominates More Than 2 Systems

Example:

- Rogue dominates gold + accessibility
- Rogue should not also dominate boss fights

### Rule 2: Every Class Has A Weakness

| Class   | Weakness                |
| ------- | ----------------------- |
| Warrior | Time + fatigue demand   |
| Ranger  | Lower burst strength    |
| Mage    | Slower peak performance |
| Rogue   | Lower ceiling           |

### Rule 3: All Content Must Be Beatable By All Classes

- No mission requires a specific class
- No boss requires a specific build
- Classes change difficulty, not access

### Rule 4: Effort Always Wins Over Class

A strong user should always outperform:

> A weak user on a "better" class

| Warrior | 140-220  |
| Ranger  | 140-220  |

| Mage    | 20-35      |

Rogue leads, but not double.

| ------- | --------------------- |

Warrior has a slightly higher ceiling, Rogue slightly lower.

## Scaling Balance (Early Vs Late Game)

### Early Game (Levels 1-5)

| Class   | Strength                        |
| ------- | ------------------------------- |
| Rogue   | Strongest, onboarding advantage |
| Mage    | Strong, consistency             |
| Ranger  | Moderate                        |
| Warrior | Weakest, learning curve         |

### Mid Game (6-10)

| Class       | Strength |
| ----------- | -------- |
| All classes | Balanced |

### Late Game (11-15)

| Class   | Strength                |
| ------- | ----------------------- |
| Warrior | Strongest ceiling       |
| Ranger  | Strong map control      |
| Mage    | Strongest survivability |
| Rogue   | Strongest economy       |

## Mechanic-Specific Balancing

### Heavier Lift Missions

- Warrior: bonus XP
- Ranger: neutral
- Mage: slight penalty reduction
- Rogue: optional bypass via abilities

### Travel System

- Ranger: fastest progression
- Rogue: gold-based flexibility
- Mage: neutral
- Warrior: slowest by design

### Boss System

- Warrior: performance-based advantage
- Ranger: prep-based advantage
- Mage: consistency-based advantage
- Rogue: efficiency-based advantage

## Anti-Exploitation Rules

### 1. Bonus Caps

- Max total modifiers: +5
- Max percentage bonuses: 15-20%

### 2. Diminishing Returns

- Stacking the same type of bonus reduces impact

### 3. Frequency Limits

- Daily abilities: once per day
- Weekly abilities: once per week

## Class Switching (Optional System)

To maintain balance and reduce regret:

### Rule

- Users can switch class, with cooldown or cost

### Benefit

- Prevents drop-off
- Allows experimentation

## Testing Guidelines

Regularly test:

### 1. Equal Effort Test

- Same workout across classes -> similar rewards

### 2. Strength Scenario

- Heavy workout -> Warrior leads

### 3. Cardio Scenario

- High steps -> Ranger leads

### 4. Short Session Scenario

- Quick workout -> Rogue leads

### 5. Consistency Scenario

- Full + streak -> Mage leads

## Failure Signals (Watch These)

If you see:

- One class used by more than 50% of users
- One class dominating boss success
- Gold inflation from Rogue
- Warrior burnout or drop-off

You need rebalance.

## Final Balance Goal

Each user should feel:

> "My class fits how I want to play"

Not:

> "This is the best class"

## Summary

Your class system is successful when:

- All classes feel different
- All classes feel viable
- Users choose based on identity, not advantage
